# 【摸鱼】模块开发日记

### 简述

投票项目已经基本开发完成，编辑、删除功能在用户量不大的情况下并不必要。大家过了开始的那阵子并没有兴趣注册一个平台继续无聊的投票。
游戏、交互游戏应该是一个比较好的模块来保持用户活跃，于是有了该项目。

### 需求、设计记录

1. 【摸鱼】分为【游戏】【房间】两个模块，房间依赖游戏
2. 可以增加【积分】，以保持用户活跃
3. 游戏表默认两个字段【题目】、【答案】用于存储默认游戏，增加【类型】字段，用于判断游戏类别，以兼容某些其他结构的游戏数据、及对数据进行相关处理

#### 表结构

```sql

# 基础游戏项目表
CREATE TABLE t_fishing_game
(
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(24),
    type VARCHAR(10) NOT NULL,
    content TEXT COMMENT '游戏内容',
    answer VARCHAR(100) COMMENT '游戏答案',

    creator      BIGINT       NOT NULL COMMENT '创建者ID',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    modifier     BIGINT,
    updated_time DATETIME,
    deleted_time DATETIME COMMENT '删除时间，NULL 表示未删除'
);
CREATE INDEX idx_fishing_game_name ON t_fishing_game (name);

# 房间好像没有持久化的必要
```
